# Task: 构建「北京中考浮力压强动态认知系统 V2」

项目代号：Physics Cognitive Runtime

## 项目定位

本项目不是传统题库系统。目标不是刷题、背公式、讲答案，而是构建一个“动态物理认知系统”，帮助初中生建立浮力、压强、液面变化、力传递、电子秤、传感器、状态变化之间的动态系统思维。

## 核心学习闭环

每个模型页面必须按照：

```txt
生活现象
→ 动态实验
→ AI提问
→ 知识点讲解
→ 数学抽象
→ 模型归类
→ 北京原题训练
→ 错因分析
→ 变式迁移
→ 知识树强化
```

禁止直接甩公式、直接给答案、纯文字解析。

## 第一阶段范围

仅实现北京中考「浮力压强专题」，不扩展其他物理专题、AI聊天、后端、登录系统。第一阶段重点是跑通“认知闭环”。

## 技术栈

- React
- TypeScript
- Vite
- TailwindCSS
- SVG 动画优先
- 不使用复杂物理引擎
- 不做真实流体模拟

原则：动画必须符合初中物理规律，而不是游戏效果。

## 四大核心模型

### Model A：漂浮 / 悬浮 / 沉底

核心知识：

```txt
漂浮：F浮 = G
悬浮：F浮 = G 且完全浸没
沉底：G = F浮 + F支
```

必须强调：漂浮静止时浮力等于重力，不是浮力大于重力。

### Model B：电子秤 / 溢水杯 / 力传递

核心知识：

```txt
电子秤本质：测的是最终传给桌面的压力
普通容器模型：Delta F电子秤 = F浮
溢水杯模型：G溢出 = F浮，电子秤示数不变
```

必须实现普通杯 / 溢水杯一键切换。

### Model C：液面变化 / 液体压强

核心知识：

```txt
p = rho g h
```

必须强调压强只看密度和深度，与水总质量、容器形状无关。

### Model D：传感器 / 自动控制 / F-h 图像

核心知识：

```txt
水位上升 → 浸入体积变大 → 浮力变大 → 拉力变化 → 传感器变化
```

必须实现 F-h 图像联动和图像悬停显示物理状态。

## V2 新增核心系统

### Liquid Level Compensation

新增 `physics/liquidLevel.ts`，实现：

```ts
calcLiquidRise({
  displacedVolume,
  containerArea
})
```

公式：

```txt
Delta h = V排 / S容器
```

系统必须实时修正液面高度、浸入深度、浮力、压强。

### Continuous Drag Runtime

禁止下一步按钮和跳帧动画。必须支持鼠标拖动物体，物体位置实时变化。每一帧实时重算浸入体积、液面、浮力、拉力、支持力、压强、图像。

### Variable Timeline

新增 `VariableTimeline.tsx`，实时绘制浮力、拉力、电子秤、水深、压强随时间变化曲线。

### Cognitive Conflict Engine

学习必须先让学生预测，再播放演示。例如：

```txt
浸没后继续下沉，浮力会：
[ 变大 ] [ 不变 ] [ 变小 ]
```

### Wrong Mental Model Detection

学生错题时必须识别错误模型，例如“浸没越深浮力越大”，并指出混淆了“压强变大”和“浮力变大”。

### Force Flow Visualization

新增 `ForceFlowView.tsx`，动态表示谁承担了力、力如何传递、最终谁压到了电子秤。

### Mother Problem Graph

新增 `MotherProblemGraph.tsx`，建立母题与变式题的关系。

## 第一优先实现题
