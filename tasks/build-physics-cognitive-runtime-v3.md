# Task: 北京中考浮力压强动态认知系统（融合版 V3）

项目代号：Physics Cognitive Runtime

## 项目定位

本项目不是传统题库系统。目标不是刷题、背公式、看解析，而是构建一个「动态物理认知系统」，帮助学生建立：

```txt
状态变化 → 力变化 → 变量联动 → 系统受力 → 模型迁移
```

的完整物理世界观。

## 核心结构

系统分为：

```txt
Physics Runtime（物理世界）
+
Cognitive Runtime（认知引擎）
+
UI Runtime（动态交互）
```

页面采用左 60% + 右 40% 双栏布局。

左侧 Dynamic Physics World 负责动态动画、拖拽交互、状态变化、力箭头、液面变化、力流动和图像联动。

右侧 Cognitive Learning Panel 负责变量面板、知识讲解、公式高亮、认知预测、原题训练、错因分析和母题归类。

## 第一阶段范围

只做北京中考「浮力压强专题」，不扩展全物理、AI聊天、登录、后端和大规模题库。第一阶段目标是跑通完整认知闭环。


禁止游戏引擎和复杂流体模拟。动画必须符合初中物理规律。

## 核心模型

### Model A：漂浮 / 悬浮 / 沉底模型

建立漂浮、悬浮、下沉、沉底的状态认知。通过液体密度和物块密度滑块实时判定状态，并渲染重力、浮力和支持力箭头。

漂浮时高亮：

```txt
F浮 = G
```

沉底时高亮：

```txt
G = F浮 + F支
```

### Model B：电子秤 / 溢水杯 / 力传递模型

拆成普通杯吊物模型和装满溢水杯模型。

普通杯：

```txt
电子秤增加 = 浮力
```

溢水杯：

```txt
溢出水重 = 浮力
电子秤不变
```

必须提供 ForceFlowView，动态表示谁承担了力、力如何传递、最终谁压到了电子秤。

### Model C：液面变化 / 液体压强模型

建立：

```txt
压强看深度
浮力看排水
```

的分离认知。必须实现液面补偿：

```txt
Delta h = V排 / S容器
```

系统实时计算液面上升、实际浸入深度、压强变化和浮力变化。

### Model D：拉力 / 传感器 / 自动控制模型

建立：

```txt
水位 → 浮力 → 拉力 → 传感器
```

的链式系统认知。必须实现 F-h 图像联动，当前状态点随水位实时移动，并自动高亮三段：

- 未接触水：拉力恒定
- 部分浸没：浮力增大，拉力减小
- 完全浸没：浮力恒定，拉力恒定

## 第一优先实现

先完整实现专项训练第3题（吊石块 + 电子秤），因为它覆盖浮力、压强、电子秤、力传递、浸没、浮力与深度。

## 最终目标

学生学完后，脑中不再是 100 道题，而是漂浮模型、电子秤模型、溢水模型、液面模型、自动控制模型，真正建立动态物理系统思维。
